# ProposaIQ Outreach Pack

## Hard truth

- Do not market this as an AI proposal generator. That is saturated and easy to ignore.
- Do not market this as a broad trust-first proposal platform. The repo no longer ships that product.
- The only wedge worth testing here is narrow: faster Upwork replies for solo React and Next.js freelancers.
- Text generation is not the moat. Reply-rate lift and repeat usage are the test.

## Positioning

- ICP: solo React / Next.js freelancers bidding on Upwork projects
- Pain: the first line is generic, the preview gets skipped, and Connects get burned on weak openings
- Core claim: ProposaIQ finds the buyer fear, matches it to one proof point, and returns a reply-focused draft fast enough to use on live jobs
- What to stop saying: trust-first proposal drafting, clarification gates, source-backed client costs, assumptions, confidence scoring

## Channel strategy

- Default: direct outreach to target freelancers
- Secondary: comment-led distribution where freelancers already complain about low reply rates
- Avoid: Product Hunt, founder communities, generic AI directories, broad social launch threads

## 14-Day Test

1. Reach out directly to `5` target freelancers per day.
2. Ask them to run one live job through the tool, not a toy example.
3. Track whether they come back for another job within the next week.
4. Track whether any of them will pay for a starter pack.

## Message Angle

Use this:

```text
Most Upwork proposals lose in the preview because the first line sounds reusable.

I built a narrow tool for React / Next.js freelancers that does three things:
- pulls the buyer fear out of the post
- matches it to one real proof point
- writes a paste-ready reply you can send fast

If you want to test it on a live job, here is the link:
https://proposal-generator-blond.vercel.app/
```

Do not use this:

```text
AI proposal generator
trust-first drafting
asks clarification questions before it writes
source-backed project proposal
```

Those are the wrong product.

## What to Measure

- visitor to builder start rate
- builder start to generate success rate
- copy rate
- return visits
- purchase clicks
- repeat usage by outreach cohort
- number of people willing to pay after one live job

## Kill Criteria

Stop adding features if `14` days of direct outreach does not produce at least:

- `10` repeat users
- `3` people willing to pay

If you miss that bar, the wedge is still too weak.
